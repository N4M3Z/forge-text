# AGENTS.md -- forge-text

> Text processing and content analysis skills for AI coding tools. Skills covering
> text transformation, grammar correction, content analysis, and document conversion.
> Multi-provider support: Claude Code, Gemini CLI, Codex, OpenCode.

## Install / Verify

```bash
make install          # deploy skills via forge CLI and activate git hooks
make validate         # module structure and code checks
make release          # build release tarball
make clean            # remove build artifacts
```

## Project Structure

```
skills/                   skill directories (SKILL.md + SKILL.yaml each)
.claude-plugin/
  plugin.json             Claude Code plugin manifest
defaults.yaml             Module config stub (forge CLI embeds provider defaults)
config.yaml               User overrides (gitignored)
module.yaml               Module metadata (name, version)
Makefile                  install, validate, release, clean via forge CLI
```

Platform directories (`.claude/`, `.gemini/`, `.codex/`, `.opencode/`) are generated by `make install` and gitignored. Source of truth is `skills/`.

## Skills

| Skill | Purpose |
|-------|---------|
| **CleanText** | Strip URLs, HTML tags, and noise from text |
| **Emojify** | Add context-appropriate emojis at natural break points |
| **ExplainSimply** | Rewrite complex content in simple, accessible language |
| **FixGrammar** | Fix grammar and spelling, preserve everything else |
| **GenerateGlossary** | Create an alphabetical glossary of key terms |
| **GenerateOutline** | Generate a hierarchical outline from any document |
| **HighImpactChanges** | Identify highest-impact changes in a document |
| **MakeLonger** | Expand text to roughly twice its length |
| **MakeShorter** | Condense text to roughly half its length |
| **MarkdownExtract** | Batch-convert a folder of source documents to canonical markdown |
| **Pandoc** | Convert documents between formats via pandoc |
| **RewriteAsTweet** | Rewrite content as a tweet or tweet thread |
| **Translate** | Translate text into any target language |

### Provider Routing

All skills deploy to all providers. The forge CLI's embedded defaults define provider targets; `config.yaml` overrides per user.

## Skill File Convention

Each skill directory contains:
- `SKILL.md` -- AI instructions with YAML frontmatter (name, description, version)
- `SKILL.yaml` -- sidecar metadata (sources, provider-specific config)
- Optional `sample.md` demo files

## Configuration

- `defaults.yaml`: module config stub (committed)
- `config.yaml`: user overrides (gitignored), same structure as defaults
- `module.yaml`: module metadata (name, version, description)

## Development Conventions

- **Skill naming**: PascalCase directories (`FixGrammar/`), matching `name:` in SKILL.md frontmatter
- **Config override**: `config.yaml` (gitignored) overrides `defaults.yaml`
- **Deployment**: [forge CLI](https://github.com/N4M3Z/forge-cli) assembles and deploys skills per provider

## Git Conventions

Conventional Commits: `type: description`. Lowercase, no trailing period, no scope.

Types: `feat`, `fix`, `docs`
