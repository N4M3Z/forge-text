# CLAUDE.md

> `CLAUDE.md` and `AGENTS.md` are autogenerated by `/Init`. Do not edit directly -- run `/Update` to regenerate.

Text processing and content analysis skills for AI coding tools.

## Install / Verify

```bash
make install          # deploy skills (Claude, Gemini, Codex, OpenCode)
make verify           # check skills deployed across all providers
make clean            # remove previously installed skills
make test             # validate-module convention checks
make lint             # mdschema + shellcheck
```

## Skills

| Skill | What it does |
|-------|-------------|
| **CleanText** | Strip URLs, HTML tags, and noise from text |
| **Emojify** | Add context-appropriate emojis at natural break points |
| **ExplainSimply** | Rewrite complex content in simple, accessible language |
| **FixGrammar** | Fix grammar and spelling, preserve everything else |
| **GenerateGlossary** | Create an alphabetical glossary of key terms |
| **GenerateOutline** | Generate a hierarchical outline from any document |
| **HighImpactChanges** | Identify the 10 highest-impact changes in a document |
| **MakeLonger** | Expand text to roughly twice its length |
| **MakeShorter** | Condense text to roughly half its length |
| **Pandoc** | Render markdown documents to PDF via pandoc |
| **RewriteAsTweet** | Rewrite content as a tweet or tweet thread |
| **Translate** | Translate text into any target language |

## Architecture

```
skills/      — 12 skill directories (SKILL.md + SKILL.yaml each)
lib/         — git submodule -> forge-lib (Rust binaries)
defaults.yaml — skill roster + provider models
config.yaml  — user overrides (gitignored), same structure as defaults
```

Platform directories (`.claude/`, `.gemini/`, `.codex/`, `.opencode/`) are generated by `make install` and gitignored. Source of truth is `skills/` and `defaults.yaml`.

## Conventions

- Explain before acting. Show what you'll do and ask for confirmation before making changes.
- Keep responses concise. One clear paragraph beats three vague ones.

## Git

Conventional Commits: `type: description`. Lowercase, no trailing period, no scope.
